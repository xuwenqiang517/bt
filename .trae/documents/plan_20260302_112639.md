## 计划：生成 Agent 规范文档（修正版）

基于完整分析，生成规范文档，包含：

### 1. 项目概述
- 量化回测系统，多策略参数优化

### 2. 目录结构与职责
| 文件 | 职责 | 备注 |
|------|------|------|
| bt2.py | 主入口 | 批量/单次回测 |
| chain.py | 策略编排 | 多进程执行 |
| strategy.py | 核心策略 | 选股/买入/卖出 |
| stock_data.py | 股票数据 | 数据获取+缓存 |
| stock_picker.py | 实时选股 | 复用strategy逻辑 |
| stock_calendar.py | 交易日历 | 日期管理 |
| param_config.py | 参数配置 | 13个参数范围 |
| param_generator.py | 参数生成 | 参数组合迭代 |
| dto.py | 数据模型 | HoldStock等 |
| local_cache.py | 本地缓存 | Feather/Parquet |
| logger_config.py | 日志配置 | 带颜色输出 |

### 3. 核心架构（修正）
- **分层架构**：入口 → 编排 → 策略 → 数据 → 基础设施
- **缓存机制**：仅运行时缓存（进程内），**无断点续跑**
- **性能优化**：Numba JIT + 两级选股缓存 + 多进程

### 4. 策略参数格式
```
持仓数量|连涨天数下限,上限,3日涨幅下限,上限,5日涨幅下限,上限,当日涨幅上限|排序方向|止损率,持仓天数,目标涨幅,回撤率
示例: 1|2,-1,7,-1,14,-1,3|0|-10,5,12,6
```

### 5. 编码规范
- 函数加注解
- 极简主义
- 避免破坏核心逻辑（缓存、Numba函数、参数格式）

### 6. 注意事项
- **无断点续跑**：大参数搜索时需一次运行完成
- 缓存是进程级的，多进程时每进程独立
